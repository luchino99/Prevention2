Data retention overrides

Each window controls how long a class of records is kept before the daily cron (fn_retention_prune, migrations 003+015) can prune it. Leave a field empty to inherit the platform default — that is the recommended setting unless a regulator or controller requires a tighter window.

Per-tenant retention is active. The cron worker reads each value below and prunes that tenant's audit / alerts / notification rows on the configured window. Empty fields fall back to the platform defaults (audit 3650, alerts-resolved 365, notifications 90, reports 730). Cron run reports include a per-tenant breakdown in the retention.run structured log.
